架構與繪圖方式分析
四層技術關係

Python (Flask + MapState)
   ↓ 呼叫
Folium (Python 套件，Leaflet 的 Python 包裝器)
   ↓ 生成
HTML 文件 (包含內嵌的 JS/CSS)
   ↓ 瀏覽器解析執行
Leaflet.js (實際在瀏覽器中渲染互動地圖)
-----------------------------------------------
1. Python 層 — 資料準備與狀態管理
map_state.py 中的 MapState 類別儲存三種核心資料：

self.markers — 艦船位置（座標、彈窗、顏色、形狀）
self.lines — 攻擊指派線（起點、終點、顏色、粗細）
self.tracks — 艦船航跡（座標序列、顏色）
這些資料由 map_service.py 的 MapService 填入，資料來源是前端 API 請求經 LLM 解析後的結果。
-----------------------------------------------
2. Folium 層 — Python 到 HTML 的橋樑
MapState.create_map() 方法（map_state.py:89 起）是核心生成邏輯：

建立底圖：folium.Map(location, zoom_start, tiles) — 這會自動生成包含 Leaflet 初始化的 HTML
內嵌 milsymbol.js：整個軍用符號庫直接寫入 HTML（離線可用）
用 Folium 原生 API 加入部分元素：
folium.Marker() → 生成 L.marker()
folium.PolyLine() → 生成 L.polyline()（用於航跡）
用 Element() 注入自訂 JavaScript：處理 Folium 無法原生支援的功能
-----------------------------------------------
3. JavaScript 注入 — 繪圖的核心手法
你選取的程式碼（靜態攻擊線）展示了關鍵的繪圖模式：

Python 不直接畫線，而是：


Python 收集線條資料 → JSON 序列化 → 嵌入 JavaScript → 注入 HTML
具體流程：

Python 將 self.lines 轉成 JSON（static_lines_js_data）
用 f-string 模板生成一段 <script>，把 JSON 嵌進去
透過 m.get_root().html.add_child(Element(...)) 注入到 Folium 生成的 HTML 中
瀏覽器載入後，這段 JS 才真正呼叫 Leaflet API 來畫線：
L.polyline() 畫攻擊線
L.marker() + SVG 畫箭頭
Math.atan2() 計算箭頭旋轉角度
為什麼不用 Folium 原生的 PolyLine？ 因為需要自訂箭頭、動態查找地圖物件、以及支援動畫控制——這些超出 Folium 的能力範圍。
-----------------------------------------------
4. Leaflet 層 — 瀏覽器端渲染
最終生成的 HTML 被存為 /maps/wta_map_*.html，前端用 iframe 載入。瀏覽器執行順序：

Leaflet 初始化地圖 → L.map()
載入 OpenStreetMap 圖磚
延遲 500ms 後執行注入的腳本：
drawMilSymbol() — 用 milsymbol 將 SIDC 代碼轉為軍用符號 SVG
drawStaticAttackLines() — 用 L.polyline() + SVG 箭頭畫攻擊線
動畫控制器（如啟用）— 用 requestAnimationFrame() 做飛彈飛行動畫
-----------------------------------------------
繪圖方式總結
元素	繪製方式	為什麼
底圖/圖磚	Folium 原生 (folium.Map)	Folium 直接支援
航跡線	Folium 原生 (folium.PolyLine)	簡單折線，Folium 夠用
艦船符號	JS 注入 (milsymbol → SVG)	需要 MIL-STD-2525 標準符號
攻擊線+箭頭	JS 注入 (L.polyline + SVG)	需要自訂箭頭和互動
飛彈動畫	JS 注入 (requestAnimationFrame)	Folium 不支援動畫


核心模式就是：能用 Folium 原生的就用，不能的就把資料序列化成 JSON 注入到 JavaScript 中，讓 Leaflet 在瀏覽器端直接處理。
